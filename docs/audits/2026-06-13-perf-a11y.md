## Detailed findings

### Performance

**P0-1, forced 1.2s loading screen (src/app/page.tsx line 361 to 368)**

```ts
useEffect(() => {
  const timer = setTimeout(() => setIsLoaded(true), 1200)
  return () => clearTimeout(timer)
}, [])
if (!isLoaded) return <LoadingScreen />
```

This is a synthetic delay, not waiting on real data. It guarantees a 1.2s LCP penalty on every first visit and makes Core Web Vitals fail. The festival site lives on shared 3G in Cape Town. Recommended fix, remove the gate entirely or gate it on a real signal (image preload, font load).

**P0-2, Unsplash images, no sizes, no priority, no optimisation contract**

Twelve `images.unsplash.com` URLs are hardcoded in the `IMAGES` map (page.tsx line 30 to 43). They render through `<Image fill>`. Without a `sizes` attribute, Next.js logs a warning and serves the largest variant. Without `priority` on the LCP image (the hero background), the browser deprioritises it. Confirm `next.config.ts` lists `images.unsplash.com` in `remotePatterns` or these images bypass optimisation entirely.

**Recommended fix**, add `sizes="(max-width: 768px) 100vw, 50vw"` per card, `priority` on the first above-the-fold image (likely the hero), and verify `remotePatterns` config.

**P1-7, raw `<img>` in vendor drawer (sectors-section.tsx line 416)**

```jsx
// eslint-disable-next-line @next/next/no-img-element
<img src={vendorBio.logo_url} alt={...} className="w-24 h-24..." />
```

Raw img with Tailwind class for dimensions, but no `width` and `height` HTML attributes. Browser reserves zero space until the image loads, then jolts the panel down. Use `<Image>` or add `width={96} height={96}` literally.

**P1-11, useEffect chain in VendorDrawerPanel**

Four effects fire in sequence on drawer open, scroll, list-fetch, bio-fetch on selection, scroll-to-bio. If a vendor rage-clicks two sectors quickly, the in-flight fetch is not aborted, only flagged with `cancelled`. AbortController would actually cancel the network call. Low impact in practice, but each click costs JSON.

### Accessibility

**P0-3, hamburger and close buttons have no aria-label (page.tsx line 442, 197)**

The mobile nav hamburger and the contact modal close X button render only icons. A screen reader announces "button" with no label.

```jsx
<button onClick={() => setMobileMenuOpen(!mobileMenuOpen)} className="lg:hidden ...">
  <motion.span ... /> <motion.span ... /> <motion.span ... />
</button>
```

**Fix**, add `aria-label="Open menu"` (and toggle on state), and `aria-expanded={mobileMenuOpen}`. Same on the modal X, add `aria-label="Close contact form"`.

The X in the SectorsSection drawer (line 297 to 312) already has a correct `aria-label`. Good pattern, copy it elsewhere.

**P0-4, color-only state on Yes/No segmented buttons (apply/page.tsx line 299 to 305, 421 to 426)**

```jsx
form.traded_before === opt ? 'bg-[#cd2653] text-white' : 'bg-white text-neutral-700'
```

Selected state communicated only by red background. No icon, no `aria-pressed`, no underline. Vendors with red-green deficiency see two near-identical buttons.

**Fix**, add `aria-pressed={form.traded_before === opt}` and a `<CheckCircle>` icon inside the selected button, or a ring + filled-circle bullet on the left.

**P1-9, P1-10, sub-44px tap targets**

iOS HIG and Material both put the floor at 44 to 48 device pixels. Found:

- Appliance +/- buttons, `w-8 h-8` (32px) at apply/page.tsx line 399, 402
- Modal close X, `w-8 h-8` at page.tsx line 197
- Hamburger lines `w-5 h-0.5` are inside a 40px parent, parent is OK, lines are decorative

**Fix**, bump to `w-11 h-11` (44px) at minimum, or pad the button so the touchable area is 44px even if the visual is smaller.

**P1-5, P1-6, no prefers-reduced-motion guard anywhere**

Multiple infinite-loop framer-motion animations:

- `LoadingScreen` orbs (page.tsx line 305 to 315), `repeat: Infinity`
- Loading bar shimmer (line 343 to 349), `repeat: Infinity`
- Sectors section background orbs (sectors-section.tsx line 528 to 553), `repeat: Infinity`
- Hero gradient pulse (page.tsx line 429 to 432), `animate-pulse`
- Marquee, infinite scroll inside `TextMarquee`

None check `useReducedMotion()` from framer-motion. Vestibular-disorder users get continuous motion they cannot turn off.

**Fix**, wrap each infinite animation in `const reduce = useReducedMotion()` and pass `animate={reduce ? {} : {...}}`. Tailwind also offers `motion-safe:` and `motion-reduce:` variants for the CSS `animate-pulse` and `animate-spin` calls.

**P2-13, honeypot announced by screen readers (apply/page.tsx line 234 to 246)**

```jsx
<div aria-hidden="true" style={{ position: 'absolute', left: '-10000px' ... }}>
  <label htmlFor="company_website_url">Company website (leave blank)</label>
  <input type="text" id="company_website_url" ... />
</div>
```

`aria-hidden` on the wrapper helps, but some screen readers (NVDA + Firefox combo) still pick up the input. Also add `aria-hidden="true"` directly on the input and `autoComplete="off"` is already correct.

### Form labels

Apply page form inputs all have correctly associated `<label>` elements above the inputs (lines 257, 275, 284, 290, 315, 321, 327). Good.

Contact modal inputs in page.tsx use `<label className="...">Name</label>` without `htmlFor`, and the input has no `id`. The label is visual only, not programmatically associated. Lines 213 to 234. Screen readers will announce the input as "edit text" with no label.

**Fix**, add `htmlFor="contact-name"` and `id="contact-name"` (and similar for email, subject, message).

## Recommendations, prioritised

1. **Kill or gate the 1.2s LoadingScreen** in page.tsx. Single biggest perf win.
2. **Add sizes + priority to Unsplash `<Image>` calls**, especially the hero and CTA backgrounds.
3. **Add aria-label to mobile hamburger, contact modal close, social icons in footer.**
4. **Add `aria-pressed` and an icon to Yes/No segmented buttons** in apply.
5. **Wrap infinite framer-motion animations in `useReducedMotion`** across page.tsx, sectors-section.tsx, smile-fm-banner if present.
6. **Bump +/- and X close buttons to 44px** minimum.
7. **Associate contact modal labels with inputs** (htmlFor + id).
8. **Convert raw `<img>` vendor logo to `<Image>` with width/height** in sectors-section.tsx.
9. **Convert portal StatTile and step `<a>` to `<Link>`** for prefetching.
10. **Add AbortController to VendorDrawerPanel fetches** to cancel stale requests on rapid sector clicks.
